## How the second brain is wired

The agent operates under two separate prompt files at the repo root:

- **`CLAUDE.md`** — loaded for normal developer sessions. Pure dev-mode rules (coding conventions, gotchas, performance baselines, pipeline mental model). **The agent is explicitly forbidden from touching `vault/` during dev sessions.**
- **`CLAUDE.agent.md`** — loaded only when the session prompt contains `[AGENT MODE]`. Defines the agent's constitution: confidence/inference rules, vault structure, tag taxonomy, sprint context, and the crawl checklist for Hub / Console / Kiosk.

This split means dev sessions and knowledge-vault sessions can't bleed into each other. Dev mode never writes vault notes; agent mode never edits source code.

## Run modes

The agent supports four run modes (configured per invocation):

| Mode | When | What it does |
|------|------|--------------|
| `baseline` | Once, before Sprint 1 | Builds the frozen pre-sprint snapshot in [[00-pre-sprint-baseline/_index\|00-pre-sprint-baseline]] |
| `open-sprint` | Start of each sprint | Creates the sprint folder, lists stories, sets up scope |
| `close-sprint` | End of each sprint | Locks the sprint folder, fills in what shipped, carries over unfinished work |
| `incremental` | Anytime | Updates living architecture notes, slices, decisions based on current code state |

Every run writes a fresh `_meta/last-run.md` so the next run knows what the last one did.
